abortPairing

(
  • device
)

Available since 1.0

Close the remote and local sockets that have been opened for pairing for the given device. If pairing is in process, this will stop it, otherwise it won't do anything. This should be implemented in a setTimeout (or similar) in the application to prevent the BluetoothServerSocket.accept() call to continue indefinitely, which would rapidly drain device power.

Parameters:

  • device String

    The name of the device with which to terminate pairing, as returned in the "bluetooth:discovery" event.

Returns:

findDevices

()

Available since 1.0

Start the remote device discovery process. From android developer reference: "The discovery process usually involves an inquiry scan of about 12 seconds, followed by a page scan of each new device to retrieve its Bluetooth name." Devices found will be returned asynchronously via the "bluetooth:discovery" event.

Returns:

void

getDeviceServices

(
  • deviceName
)

Available since 1.0

Retrieve the names of services that the remote device may use to connect. These are returned asynchronously via the "bluetooth:services" event. You must pass the name of a service that the remote device supports in order to pair with it. Unless there is a particular service that you need to connect to, you will probably have the most success just using "Serial Port".

Parameters:

  • deviceName String

    The name of the device to query, as returned in the "bluetooth:discovery" event.

Returns:

pairDevice

(
  • deviceName
  • serviceName
)

Available since 1.0

Initiate a connection to the remote device on the specified channel. In some cases you must confirm this connection on the remote device, or with a PIN, and you will be prompted to do so. After pairing is complete, data transfer may begin immediately. The event "bluetooth:paired" is fired when pairing has completed successfully.

Parameters:

  • deviceName String

    The name of the device with which to pair, as returned in the "bluetooth:discovery" event.

  • serviceName String

    The name of the remote service (channel) to connect to as returned in the "bluetooth:services" event.

Returns:

setReadSize

(
  • bytes
)

Available since 1.1

Optionally set the size of the output buffer. Default is 1024 bytes. This is the maximum size of the Titanium.Blob returned in each "bluetooth:data:deviceName" event. New buffer size applies only to connections made in the future. So in order to reconfigure the output buffer of a currently active connection, you would need to disconnect the device, set the buffer size, then reconnect. Ideally this is set once, after starting the service, but it can be configured differently for each connection by calling setReadSize(bytes) before calling pairDevice.

Parameters:

  • bytes Number

    Max size of data returned in each "data" event. Defaults to 1024 bytes.

Returns:

void

startService

()

Available since 1.0

Start the bluetooth service controlled by the module. This is called by the constructor, so it shouldn't be necessary to call it unless the service is stopped manually. It is safe to call this method while the service is running. In that case, it will have no effect. Calling any other methods while the service is not running will result in an error.

Returns:

void

()

Available since 1.0

Manually stop the bluetooth service. If you don't call this when your app exits, the service will continue running in the background!

Events

bluetooth:data:deviceName

Available since 1.0

Transmits a package of data from the remote device to the host device. Once a connection is established you should begin listening for this event. The event name includes the name of the device in place of "deviceName".

Event Payload:

  • data Titanium.Blob

    Binary data transmitted from the remote device.

bluetooth:destroy

Available since 1.2

Fires when the module activity is destroyed. Receives the native android onDestroy event. Be sure to understand the android activity lifecycle before attempting to use. See http://developer.android.com/reference/android/app/Activity.html

bluetooth:discovery

Available since 1.0

Event Payload:

  • devices Object

    Keys are device names and values are hardware addresses. Currently, only the name is used by this module.

bluetooth:error

Available since 1.0

Identical to bluetooth:message, but more or less important, depending on your perspective.

Event Payload:

  • message String

    Something's gone terribly wrong.

bluetooth:message

Available since 1.0

Occasionally the module will have something to say. Listen for it on this event.

Event Payload:

  • message String

    A message from the bluetooth module.

bluetooth:paired

Available since 1.0

Fires after successfully pairing with device.

Event Payload:

  • device String

    The name of the device that is now paired.

bluetooth:pause

Available since 1.2

Fires when the native android onPause event is received by the module. Be sure to understand the android activity lifecycle before attempting to use. See http://developer.android.com/reference/android/app/Activity.html

bluetooth:resume

Available since 1.2

Fires when the native android onResume event is received by the module. Be sure to understand the android activity lifecycle before attempting to use. See http://developer.android.com/reference/android/app/Activity.html

bluetooth:services

Available since 1.0

NB. When queried by the service discovery protocol, devices return the UUIDs of services that they claim to support. For the sake of human readability, these UUIDs have been mapped to names/descriptions, which is what this module uses to designate a service with which to connect. The names should not necessarily be relied upon as accurate. If you need to know the UUID, the mapping comes from here: http://bluetooth-pentest.narod.ru/doc/assignednumbers-servicediscovery.html

Event Payload:

  • services Array

    Array of the names of services that the device claims to support.

bluetooth:start

Available since 1.2

Fires when the native android onStart event is received by the module. Be sure to understand the android activity lifecycle before attempting to use. See http://developer.android.com/reference/android/app/Activity.html

bluetooth:stop

Available since 1.2

Fires when the native android onStop event is received by the module. Be sure to understand the android activity lifecycle before attempting to use. See http://developer.android.com/reference/android/app/Activity.html
